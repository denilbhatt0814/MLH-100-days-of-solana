## 📝 Overview

Day 3 focuses on using the Solana CLI to set up a wallet, check balances, and extract transaction details programmatically. The challenge demonstrates how to automate wallet management and transaction analysis using bash scripting. 🖥️💸

---

## 🛠️ Steps Followed

1. <b>Solana CLI Installation & Wallet Setup</b> 🛠️
   - Installed the Solana CLI using the official installer script.
   - Verified installation and updated the PATH if needed.
   - Generated a new CLI wallet and set the network to Devnet.
   - Displayed the wallet address for reference.

2. <b>Balance and Transaction Script</b> 💰
   - Created a script to fetch and display the wallet's balance in both SOL and Lamports.
   - Queried the most recent transaction hash for the wallet.
   - Extracted the transaction fee from the confirmation output, converting it from SOL to Lamports and removing any special symbols for clean display.

---

## ✨ Key Takeaways

- <b>CLI Automation</b> 🤖: Automated wallet setup and transaction analysis using bash scripting.
- <b>String Parsing</b> 📝: Used `sed`, `awk`, and `bc` to parse and format CLI output.
- <b>Unit Conversion</b> 🔄: Practiced converting between SOL and Lamports (1 SOL = 1,000,000,000 Lamports).
- <b>Robust Scripting</b> 🛡️: Handled special symbols (like ◎) in CLI output for accurate calculations.
